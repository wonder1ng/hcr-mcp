# hcr-mcp 뉴스 수집기(`company_report/news/collector.py`) 개선 기록

`collect_recent_issues("윕스")`로 실제 검증하면서 발견·수정한 문제들. 최신 항목이 영향이 가장 컸음.

## 가장 결정적이었던 수정: 검색 URL 파라미터 오류

**증상**: 완전히 무관한 기사(축구 선수 해트트릭, 증권사 WTS 분쟁, 전고체 배터리 산업 뉴스 등)가
"윕스" 이슈로 다수 섞여 나옴. 본문 기반 필터링/임베딩을 아무리 정교화해도 근본적으로 안 없어짐.

**원인**: `_date_range_params`가 실제 검증된 값과 다른 파라미터를 쓰고 있었음.
- `sort=0` (관련도순) → 올바른 값은 `sort=1` (최신순)
- `nso=so:r,...` → 올바른 값은 `nso=so:dd,...`

사용자가 직접 검증한 레퍼런스 URL과 대조해서 발견. 검색 자체가 느슨한 매칭으로 결과를 반환하고
있었던 것 — 이후 단계(그룹핑, 필터링, medoid 선정)가 아무리 정교해도 애초에 검색 결과 풀 자체가
오염되어 있으면 소용없었음.

**효과**: 완전 무관 기사가 전부 사라짐. 오히려 이전엔 못 찾던 진짜 중요한 이슈("K-특허 최대 거함
'윕스號', 어디로?...잇단 송사에, CEO 리스크까지" — 소송 패소, 직원 100여명 권고사직)가 새로
발굴됨. **검증 없이 "실증 확인"이라고 남겨둔 상수값도 주기적으로 재검증할 필요가 있음을 보여준
사례.**

## 그 외 수정 사항 (영향이 작은 순서 아님, 시간순)

1. **무한루프 버그**: 좁은 날짜 구간에서 네이버가 "검색결과 없음" 페이지를 정상 응답(200)으로
   줄 때 스로틀링으로 오판해 무한 재시도. `#notfound` 구조적 요소로 확정 판별하도록 수정.
2. **검색 라운드 스케줄**: 365일 단순 롤링(최대 5년) → 6개월/캘린더 연도 기준(6개월→올해
   나머지→작년→재작년→4/1 이전이면 그전해)으로 재작성. 겹침 없이 검증.
3. **요약/분류 all-or-nothing 폴백**: 배열 길이가 하나라도 안 맞으면 배치 전체가 빈 값으로
   버려지던 버그 → 인덱스 명시 구조화 출력(langchain `with_structured_output`)으로 교체,
   요약+분류 한 번의 호출로 통합.
4. **무관 기사 혼입 + 근접 중복 미병합**: 임베딩 기반 사전 클러스터링(제목+본문, leader
   clustering) 추가, 회사명이 title/본문 어디에도 없으면 결정론적으로 이슈 제거하는 안전망 추가.
5. **과잉 병합**: LLM에게 이미 만든 그룹을 다시 병합시키는 재귀 패스가 무관한 이슈끼리도 과도하게
   합쳐버림 → 제거, 병합은 앞단 임베딩 클러스터링만 담당.
6. **요약 배치 컨텍스트 로트**: 기사 본문 전체를 25개씩 한 번에 요약 요청하니 뒤쪽 항목 내용을
   놓치고 같은 문장을 반복 → 배치 크기를 6개로 축소(그룹핑용 25개와 분리).
7. **`_MIN_ISSUES` 5→15**: 얕은 6개월 라운드에서 조기 종료되던 것을 방지, 더 깊은 과거까지 탐색.
8. **시간대 연결 + 기업 연관도 재평가**: 별도 LLM 패스로 (a) 후속/갱신 이슈를 타임라인으로
   연결(병합 문장은 LLM이 새로 안 쓰고 기존 gist를 결정론적으로 이어붙임 — 환각 방지), (b) 사건
   유형별 고정 중요도가 아니라 실제 내용(확정 사실/추측성, 당사자/단순언급, 전사적/지엽적)을
   기준으로 중요도 재평가(taxonomy 기준값을 앵커로 제공 + reasoning 근거 필수).
9. **날짜/제목 대표 기사 통일**: 그룹 내 서로 다른 기사에서 제목과 날짜를 따로 뽑아 불일치가
   생기던 버그 → 항상 같은(가장 이른) 대표 기사에서 함께 추출.
10. **본문 우선 수집 + medoid 대표 선정**: 검색 직후(그룹핑 전) 전체 원본 기사 본문을 미리
    확보, 언론사 등장 순서가 아니라 본문 임베딩 기준 medoid(그룹 내 평균 유사도 최고)로 대표
    기사 선정.
11. **임베딩 중복 병합 윈도우 1일로 축소**: 사용자 지시로 조정(±3일 → 당일만).
12. **요약 단계 주체 오귀속(할루시네이션) 수정**: "삼성이 그리는 '로봇 기업'의 꿈" 기사 —
    윕스 관계자가 삼성전자의 투자를 코멘트한 문장인데, gist가 "윕스가 최대주주 지위를 확보"로
    행위 주체를 잘못 붙임(원문 확인으로 검증됨: 실제 행위자는 삼성전자, 윕스는 데이터 출처로만
    인용). `_IssueSummaryItem`에 `gist`보다 앞서는 `actor` 필드를 추가해 "행위자 먼저 판단 →
    그 행위자를 주어로 요약"하는 순서를 강제(구조화 출력은 필드 순서대로 생성되므로 순서
    자체가 사고 과정을 스캐폴딩함 — 타임라인 요약 연구의 "먼저 요소 추출 후 통합" 기법과 같은
    원리). 재검증 결과 오귀속 사라짐.

## 남은 이슈

- 동명이인/제품명 충돌(예: "블루맥스 윕스" WiFi 인증 = 시큐아이 제품명)은 완전히 걸러지지 않고
  낮은 중요도로 남는 경우가 있음 — 100% 보장은 어려운 영역.
- 근접 중복 병합 윈도우를 1일로 좁혀서, 하루 이상 차이나는 완전히 같은 사건의 후속 보도는 별도
  이슈로 남을 수 있음(사용자 요청에 따른 트레이드오프).
